## 📱 Основы мобильной экосистемы <a id="основы-мобильной-экосистемы"></a>

### Разновидности устройств и их характеристики <a id="разновидности-устройств"></a>

| Категория | Примеры | Ключевые характеристики | Влияние на тестирование |
|-----------|---------|------------------------|--------------------------|
| **Смартфоны** | iPhone 15, Samsung Galaxy S24, Pixel 8 | Экран 5–7", ёмкостный сенсор, аккумулятор 3000–5000 мАч | Разные разрешения, плотность пикселей (320–480+ ppi), жесты вместо кнопок |
| **Планшеты** | iPad Pro, Galaxy Tab S9 | Экран 8–13", часто с поддержкой стилуса | Адаптивный UI (Split View, Slide Over), ориентация landscape/portrait |
| **Складные устройства** | Galaxy Z Fold 5, Pixel Fold | Двойной экран, шарнир, режим Flex | Тестирование при смене состояния (сложен/раскрыт), продолжение работы после складывания |
| **Носимая электроника** | Apple Watch, Galaxy Watch | Маленький экран (1–2"), круглый/квадратный, сенсоры здоровья | UI должен быть читаем на малом экране, тестирование через companion app |
| **Автомобильные системы** | Android Auto, CarPlay | Экран авто, управление с руля/голосом | Ограниченное взаимодействие, димминг в ночном режиме, голосовые команды |

### Специфические характеристики устройств, важные для тестирования

1. **Физические кнопки и сенсоры**
   - Громкость, питание, домой (если есть) → тестирование прерываний (звонок при работе приложения)
   - Датчики: акселерометр, гироскоп, NFC, биометрия (Face ID / Touch ID / сканер пальца)

2. **Вариативность экранов**
   - Разрешения: от 640×480 (старые часы) до 2960×1440 (флагманы)
   - Вырезы: notch, punch-hole, динамический остров (iOS) — проверять, что контент не обрезается
   - Частота обновления: 60 / 90 / 120 Hz → тестирование анимаций и скролла

3. **Аккумулятор и энергопотребление**
   - Режим энергосбережения может ограничивать фоновую активность, частоту CPU, синхронизацию
   - Тестировать при 100%, 15%, 5% заряда

4. **Память и хранилище**
   - Оперативная память (2–12 ГБ) → как приложение ведёт себя при нехватке RAM (выгрузка из фона)
   - Внутреннее хранилище (32 ГБ – 1 ТБ) → установка при недостатке места, работа с кэшем

5. **Процессоры и архитектуры**
   - ARM (Snapdragon, Exynos, Tensor) vs Apple Silicon (A, M серии)
   - 32-bit / 64-bit — тестирование совместимости старых приложений

### Основные различия между типами устройств в контексте тестирования

1. **Размер экрана** влияет на компоновку UI, читаемость текста, размер кликабельных элементов.
2. **Форм-фактор** определяет возможные состояния устройства (сложен/раскрыт, портрет/ландшафт, режим планшета).
3. **Аппаратные возможности** (NFC, биометрия, сенсоры) могут отсутствовать на бюджетных устройствах — нужна проверка graceful degradation.
4. **Производительность** сильно варьируется — обязательно тестировать на слабых устройствах (старые модели, low-end).
5. **Версия ОС** часто связана с железом (например, старые iPhone не получают новую iOS) — отсюда фрагментация.
