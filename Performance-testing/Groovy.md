## 🌏 Типы данных <a id="типы-данных"></a>

### Примитивные типы и объекты <a id="примитивные-и-объекты"></a>

| Категория     | Примитивный тип | Соответствующий объект        | Пример использования   | Особенности / Различия                                             |
| ------------- | --------------- | ----------------------------- | ---------------------- | ------------------------------------------------------------------ |
| Целые числа   | `byte`          | `Byte`                        | `byte b = 10`          | 8 бит, от -128 до 127; объект `Byte` нужен для методов и коллекций |
|               | `short`         | `Short`                       | `short s = 1000`       | 16 бит, от -32768 до 32767                                         |
|               | `int`           | `Integer`                     | `int i = 42`           | 32 бит, автоупаковка при использовании как `Integer`               |
|               | `long`          | `Long`                        | `long l = 100000L`     | 64 бит                                                             |
| Дробные числа | `float`         | `Float`                       | `float f = 3.14`       | 32-битное число с плавающей точкой                                 |
|               | `double`        | `Double`                      | `double d = 3.14159`   | 64-битное число с плавающей точкой, более точное                   |
| Символы       | `char`          | `Character`                   | `char c = 'A'`         | Один символ Unicode, объект `Character` нужен для методов          |
| Логический    | `boolean`       | `Boolean`                     | `boolean flag = true`  | `true` или `false`, объект `Boolean` нужен для методов             |
| Строки        | —               | `String`                      | `String str = "Hello"` | Всегда объект, примитивного типа нет, поддерживает методы Groovy   |
| Коллекции     | —               | `List`, `Map`, `Range`, `Set` | `List l = [1,2,3]`     | Всегда объекты, динамические, мощные методы для работы с данными   |
| Универсальный | —               | `def`                         | `def x = 10`           | Неявно типизированная переменная, Groovy сам определяет тип        |

**Основные различия между примитивами и объектами в Groovy** 

1. **Примитивы** занимают меньше памяти и работают быстрее, но не имеют методов.
2. **Объекты** — полноценные экземпляры классов, имеют методы и могут использоваться в коллекциях.
3. **Автоупаковка/распаковка**: Groovy автоматически превращает примитивы в объекты и обратно при необходимости.
4. **Типизация**: с `def` можно создавать переменные без указания конкретного типа, и Groovy сам выбирает, будет это примитив или объект.

### Коллекции: List, Map, Set <a id="коллекции"></a>

| Коллекция                    | Тип              | Порядок элементов                                               | Дублирование         | Доступ                      | Изменяемость             | Пример                                           | Основные методы / особенности                            |
| ---------------------------- | ---------------- | --------------------------------------------------------------- | -------------------- | --------------------------- | ------------------------ | ------------------------------------------------ | -------------------------------------------------------- |
| **List**                     | `List`           | Сохраняется (индексируемая)                                     | Разрешено            | По индексу `list[0]`        | Изменяемый (`ArrayList`) | `def list = [1,2,3,2]`                           | `add()`, `remove()`, `get()`, `each{}`, `collect{}`      |
| **LinkedList**               | `LinkedList`     | Сохраняется                                                     | Разрешено            | По индексу, итератор        | Изменяемый               | `def ll = new LinkedList([1,2,3])`               | Быстрое добавление/удаление в середине                   |
| **Set**                      | `Set`            | Не гарантируется (`HashSet`), `LinkedHashSet` сохраняет порядок | Запрещено            | Через перебор, `contains()` | Изменяемый               | `def set = [1,2,3,2] as Set` → `[1,2,3]`         | `add()`, `remove()`, `contains()`, `each{}`              |
| **SortedSet / TreeSet**      | `SortedSet`      | Сортированный                                                   | Запрещено            | Через перебор               | Изменяемый               | `def ts = new TreeSet([3,1,2])` → `[1,2,3]`      | Автоматическая сортировка элементов                      |
| **Map**                      | `Map`            | Сохраняется порядок вставки (`LinkedHashMap`)                   | Ключи уникальны      | По ключу `map['key']`       | Изменяемый               | `def map = [name:'John', age:30]`                | `put()`, `remove()`, `each{ k,v-> }`, `collectEntries{}` |
| **SortedMap / TreeMap**      | `SortedMap`      | Сортированный по ключу                                          | Ключи уникальны      | По ключу                    | Изменяемый               | `def tm = new TreeMap([b:2, a:1])` → `[a:1,b:2]` | Автоматическая сортировка ключей                         |
| **Range**                    | `Range`          | Сохраняется (начало → конец)                                    | Разрешено            | По индексу, перебор         | Неизменяемый             | `def r = 1..5` → `[1,2,3,4,5]`                   | Можно использовать как `list` и проверять `in range`     |
| **Queue**                    | `Queue`          | Сохраняется                                                     | Разрешено            | Через методы очереди        | Изменяемый               | `def q = new LinkedList([1,2,3]) as Queue`       | `offer()`, `poll()`, `peek()`                            |
| **Deque**                    | `Deque`          | Сохраняется                                                     | Разрешено            | Через методы очереди        | Изменяемый               | `def dq = new ArrayDeque([1,2,3])`               | Двусторонняя очередь: `addFirst()`, `addLast()`          |
| **Stack**                    | `Stack`          | Сохраняется                                                     | Разрешено            | Через методы стека          | Изменяемый               | `def stack = new Stack()`                        | `push()`, `pop()`, `peek()`                              |
| **BitSet**                   | `BitSet`         | По позиции бита                                                 | —                    | По индексу                  | Изменяемый               | `def bits = new BitSet()`                        | Эффективно для хранения битов, логические операции       |
| **Range с шагом**            | `Range`          | Сохраняется                                                     | Разрешено            | По индексу, перебор         | Неизменяемый             | `def r = 0..10 step 2` → `[0,2,4,6,8,10]`        | Можно использовать в циклах и фильтрации                 |
| **Lazy коллекции / Streams** | `LazyCollection` | Сохраняется                                                     | Зависит от источника | Через итератор              | Изменяемый               | `def lz = (1..100).lazy.filter{ it % 2 == 0 }`   | Вычисляется по мере обращения (лениво)                   |

**Структура и различия**

* **Списки (List, LinkedList, Queue, Deque, Stack)** — индексируемые, сохраняют порядок, допускают дубликаты.
* **Множества (Set, SortedSet)** — уникальные элементы, могут быть неупорядоченными или сортированными.
* **Карты (Map, SortedMap)** — пары ключ → значение, уникальные ключи, сохраняют порядок или сортируют по ключу.
* **Диапазоны (Range)** — удобны для последовательностей чисел или символов, неизменяемы, поддерживают `in`.
* **Специализированные структуры (BitSet, LazyCollection)** — оптимизированы для конкретных задач, например, битовые операции или ленивые вычисления.

### Range и Spread-оператор <a id="range-spread"></a>

**1. Range (диапазоны)**

| Синтаксис | Тип | Описание | Пример |
|:----------|:----|:---------|:--------|
| `a..b` | Включительный | Включает оба конца | `1..5` → `[1,2,3,4,5]` |
| `a..<b` | Исключительный | Включает `a`, исключает `b` | `1..<5` → `[1,2,3,4]` |

**Характеристики:**
| Свойство | Описание |
|:---------|:---------|
| **Типы данных** | Числа (`IntRange`), символы (`'a'..'d'`), Comparable |
| **Реализация** | Реализует `List` (можно как список) |
| **Ленивость** | Не хранит элементы, генерирует по требованию |

**Применение:**
| Назначение | Пример |
|:-----------|:--------|
| Итерация | `for (i in 1..10) {}` |
| Срезы | `list[2..5]`, `str[0..<3]` |
| Проверка | `if (x in 1..10) {}` |
| Switch | `case 1..10:` |

**2. Spread-оператор (*.)**

| Конструкция | Действие | Возврат |
|:------------|:---------|:--------|
| `list*.method()` | Вызов метода у всех элементов | Новый `List` |
| `list*.property` | Доступ к свойству всех элементов | Новый `List` |
| `null*.anything` | Безопасная работа с `null` | `null` |

**Сравнение:**
| Без оператора | Со spread-оператором |
|:--------------|:---------------------|
| `result = []`<br>`for (e in coll) {`<br>`    result << e.name`<br>`}` | `result = coll*.name` |

**Нюансы:**
| Ситуация | Поведение |
|:---------|:----------|
| Коллекция = `null` | → `null` (нет NPE) |
| Элемент внутри = `null` | → Ошибка при вызове метода |
| Тип на выходе | Всегда `ArrayList` |

**3. Комбо**

```groovy
(1..5)*.multiply(2)      // [2,4,6,8,10]
('a'..'d')*.toUpperCase() // ['A','B','C','D']
```

### Строки и GString <a id="строки-gstring"></a>

**1. Типы строк**

| Синтаксис | Название | Особенности | Экранирование |
|:----------|:---------|:-------------|:--------------|
| `'текст'` | Одинарные | Обычный `String`, без интерполяции | `\'` |
| `"текст"` | Двойные | `String` или `GString` (если есть `${}`) | `\"` |
| `'''текст'''` | Тройные | Многострочные, без интерполяции | Нет |
| `"""текст"""` | Тройные двойные | Многострочные, **с** интерполяцией | Нет |
| `/текст/` | Slashy | Для regex, экранирование `/` не нужно | `\/` |
| `$/текст/$` | Dollar slashy | Для сложных regex, любой контент | `$` |

***Интерполяция** — это подстановка значений переменных/выражений внутрь строки.*
*Простыми словами: Вместо склеивания кусков текста через +, вы просто вставляете переменную прямо в строку, и Groovy сам подставляет её значение.*

**2. GString (интерполяция)**

| Выражение | Описание | Пример | Результат |
|:----------|:---------|:--------|:----------|
| `${variable}` | Вставка переменной | `"Hello ${name}"` | `Hello John` |
| `${expression}` | Вставка выражения | `"Sum = ${a + b}"` | `Sum = 7` |
| `$variable` | Для простых полей | `"Hello $name"` | `Hello John` |

**Важно:** GString — НЕ `String`, это отдельный класс с ленивым вычислением.

**3. Сравнение строк**

| Ситуация | Поведение |
|:---------|:----------|
| `==` | Сравнивает **значения**, а не ссылки (в отличие от Java) |
| `.equals()` | То же самое, что `==` |
| `.is()` | Проверка ссылок (как `==` в Java) |

```groovy
'abc' == new String('abc')  // true (по значению)
'abc'.is(new String('abc')) // false (разные объекты)
```

**4. Полезные методы (дополнение к Java)**

| Метод | Описание | Пример |
|:------|:---------|:--------|
| `-` | Удаление подстроки | `"test" - "es"` → `"tt"` |
| `*` | Повторение | `"ha" * 3` → `"hahaha"` |
| `[]` | Срез по диапазону | `"Hello"[1..3]` → `"ell"` |
| `[]` | Срез по индексу | `"Hello"[-1]` → `"o"` |
| `capitalize()` | Заглавная первая | `"groovy".capitalize()` → `"Groovy"` |
| `toList()` | В список символов | `"abc".toList()` → `['a','b','c']` |

**5. Многострочные строки**

| Тип | Интерполяция | Использование |
|:----|:-------------|:---------------|
| `''' ... '''` | Нет | SQL, JSON шаблоны без переменных |
| `""" ... """` | Да | Шаблоны с переменными, документация |

```groovy
def name = "John"
def json = """{
    "name": "$name",
    "age": ${25 + 5}
}"""
```

## 🔵 Методы, Closures и лямбды <a id="closures"></a>

### Определение и вызов методов <a id="методы"></a>

**1. Синтаксис определения**

| Стиль | Синтаксис | Особенности |
|:------|:----------|:-------------|
| **Явный тип** | `тип имя(параметры) { тело }` | Классический Java-стиль |
| **def** | `def имя(параметры) { тело }` | Возвращаемый тип выводится автоматически |
| **Однострочный** | `def имя(п) = выражение` | Без `return`, результат выражения |

```groovy
// Примеры
int sum(int a, int b) { return a + b }
def sum(a, b) { a + b }              // return не обязателен
def square(x) = x * x                 // Однострочная форма
```

**2. Параметры**

| Тип | Синтаксис | Описание | Пример вызова |
|:----|:----------|:---------|:---------------|
| **Обязательные** | `def f(a, b) {}` | Все параметры должны быть переданы | `f(1, 2)` |
| **Со значением по умолчанию** | `def f(a, b=5) {}` | Можно не передавать `b` | `f(1)` → `b=5` |
| **С Map (именованные)** | `def f(Map args) {}` | Имитация именованных параметров | `f(a:1, b:2)` |
| **Varargs** | `def f(a, b...) {}` | Массив переменной длины | `f(1, 2, 3, 4)` |

**3. Особенности Groovy-методов**

| Правило | Поведение |
|:--------|:----------|
| **return необязателен** | Возвращается результат последнего выражения |
| **Скобки необязательны** | `println "Hello"` вместо `println("Hello")` |
| **Точка с запятой необязательна** | Каждая инструкция с новой строки |
| **Проверка типов** | Только если указан тип, иначе `Object` (динамика) |

**4. Вызов методов**

| Ситуация | Синтаксис | Комментарий |
|:---------|:----------|:-------------|
| Со скобками | `obj.method(1, 2)` | Явный стиль |
| Без скобок | `obj.method 1, 2` | Только если нет ambiguity |
| С именованными параметрами | `obj.method(a:1, b:2)` | Параметр должен быть `Map` |
| С методом как аргументом | `list.each( { println it } )` | Можно опустить скобки у внешнего вызова |

**5. Методы vs Замыкания (важно!)**

| Характеристика | Метод | Замыкание |
|:---------------|:------|:----------|
| **Принадлежность** | Привязан к классу | Объект первого класса |
| **Переменная** | Нельзя присвоить переменной | Можно: `def c = { it * 2 }` |
| **Передача в методы** | Только ссылка `&method` | Просто передать переменную |
| **Контекст** | `this` = экземпляр класса | `this`, `delegate`, `owner` |

**6. Примеры компактно**

```groovy
// Определение
def add(a, b) = a + b

// Вызов
add 5, 3              // 8 (без скобок)
println add(5, 3)      // 8

// Параметры по умолчанию
def greet(name = "мир") = "Привет, $name"
greet()                // Привет, мир
greet("Анна")          // Привет, Анна
```

### Closure: создание и использование <a id="closures-использование"></a>

**1. Синтаксис создания**

| Стиль | Синтаксис | Особенности |
|:------|:----------|:-------------|
| **Без параметров** | `{ → тело }` | Стрелка обязательна, если нет параметров |
| **С одним параметром** | `{ it → тело }` или `{ тело }` | `it` — неявный параметр (если не указан) |
| **С несколькими** | `{ a, b → тело }` | Явное перечисление параметров |
| **Многострочное** | `{ → ` <br>`   строка1` <br>`   строка2` <br>`}` | Последнее выражение — результат |

```groovy
// Примеры
def noParams = { → "hello" }           // Без параметров
def oneParam = { it * 2 }               // С неявным it
def twoParams = { a, b → a + b }        // Два параметра
def multiLine = {                        // Многострочный
    def temp = it * 2
    temp + 1
}
```

**2. Параметры Closure**

| Особенность | Синтаксис | Пример | Поведение |
|:------------|:----------|:--------|:-----------|
| **Неявный it** | `{ it }` | `{ it.toUpperCase() }` | Доступен, если не указаны параметры |
| **Типизация** | `{ String a → }` | `{ String s → s.length() }` | Можно указывать типы |
| **По умолчанию** | `{ a, b=5 → }` | `{ a, b=5 → a + b }` | Значения по умолчанию |
| **Varargs** | `{ a, b... → }` | `{ a, b... → b.sum() }` | Переменное число |

**3. Вызов Closure**

| Способ | Синтаксис | Пример | Результат |
|:--------|:----------|:--------|:-----------|
| **call()** | `closure.call(args)` | `c.call(5)` | Вызов |
| **Как метод** | `closure(args)` | `c(5)` | То же самое |
| **Currying** | `closure.curry(a)` | `add.curry(5)(3)` | Частичное применение |
| **Композиция** | `c1 >> c2` | `(c1 >> c2)(x)` | `c2(c1(x))` |

**4. Важные свойства**

| Свойство | Описание | Доступ |
|:---------|:---------|:--------|
| **this** | Внешний класс (где создано) | `{ this }` |
| **owner** | Владелец (класс или другой closure) | `{ owner }` |
| **delegate** | Делегат (по умолчанию = owner) | `{ delegate }` |
| **maximumNumberOfParameters** | Число параметров | `closure.maximumNumberOfParameters` |
| **parameterTypes** | Типы параметров | `closure.parameterTypes` |

**5. Стратегии делегирования**

| Стратегия | Поведение |
|:----------|:-----------|
| **OWNER_FIRST** | Сначала ищет в owner, потом в delegate (по умолчанию) |
| **DELEGATE_FIRST** | Сначала в delegate, потом в owner |
| **OWNER_ONLY** | Только в owner |
| **DELEGATE_ONLY** | Только в delegate |
| **TO_SELF** | Только в самом closure |

```groovy
closure.delegate = obj
closure.resolveStrategy = Closure.DELEGATE_FIRST
```

**6. Closure как параметры (DSL)**

| Паттерн | Синтаксис | Пример |
|:--------|:----------|:--------|
| **Последний параметр** | `each { }` | `list.each { println it }` |
| **Вне скобок** | `method arg, { }` | `withStream { }` |
| **Оборачивание** | `with(obj) { }` | `new File('x').with { text }` |

```groovy
// DSL-стиль
def configure(Map cfg, Closure c) { 
    c.delegate = cfg
    c()
}

configure(width:100) { 
    height = 200      // delegate используется
}
```

**7. Методы vs Closure (сравнение)**

| Критерий | Метод | Closure |
|:---------|:------|:---------|
| **Объект** | Нет (привязан) | Да (можно передать) |
| **Возврат** | `return` из метода | `return` только из closure |
| **Параметры** | Фиксированы | Гибкие (it, defaults) |
| **Контекст** | Статический | Динамический (delegate) |
| **Композиция** | Нет | `>>`, `<<`, `curry()` |

**8. Полезные методы Closure**

| Метод | Описание | Пример |
|:-------|:---------|:--------|
| **curry()** | Фиксирует первые аргументы | `mul.curry(2)(3)` → `6` |
| **rcurry()** | Фиксирует последние | `div.rcurry(2)(6)` → `3` |
| **ncurry()** | Фиксирует по позиции | `ncurry(1, 5)(1, 2)` |
| **memoize()** | Кэширует результаты | `slow.memoize()(x)` |
| **trampoline()** | Для рекурсии | `fact.trampoline()(n)` |
| **compose()** | `<<` (до) | `(f << g)(x)` = `f(g(x))` |
| **andThen()** | `>>` (после) | `(f >> g)(x)` = `g(f(x))` |

```groovy
// Пример memoize
def fib
fib = { n -> n < 2 ? 1 : fib(n-1) + fib(n-2) }.memoize()
fib(40)  // быстро даже с рекурсией
```

### Ссылки на методы и лямбда-выражения <a id="методы-лямбда"></a>

**1. Ссылки на методы (Method Reference Operator)**

Оператор `.&` позволяет взять существующий метод и превратить его в **Closure**. Это полезно, когда нужно передать метод одного объекта в качестве аргумента другому (например, в метод `each` или `collect`).

| Синтаксис | Описание | Пример |
| --- | --- | --- |
| `объект.&метод` | Ссылка на метод конкретного экземпляра | `str.&toUpperCase` |
| `Класс.&статическийМетод` | Ссылка на статический метод класса | `Integer.&parseInt` |

**Пример использования:**

```groovy
def list = ["apple", "banana", "cherry"]
def upper = "test".&toUpperCase // создаем closure из метода строки

// Передаем ссылку на метод в collect
def result = list.collect(String.&toUpperCase) 
// Результат: ["APPLE", "BANANA", "CHERRY"]

```

**2. Лямбда-выражения (Groovy 3+)**

Начиная с версии 3.0, Groovy поддерживает синтаксис лямбд Java 8. Они очень похожи на Closure, но имеют свои технические нюансы.

| Тип | Синтаксис | Ключевая особенность |
| --- | --- | --- |
| **Лямбда** | `(a, b) -> a + b` | Более строгая, компилируется в анонимный класс (как в Java) |
| **Closure** | `{ a, b -> a + b }` | Динамическая, поддерживает `delegate` и `owner` | 

**Сравнение синтаксиса:**

```groovy
// Closure (традиционный Groovy)
list.stream().map({ it.toUpperCase() })

// Лямбда (стиль Java 8 в Groovy 3)
list.stream().map(s -> s.toUpperCase())

```

**3. Основные различия: Closure vs Lambda**

| Характеристика | Closure `{}` | Lambda `->` |
| --- | --- | --- |
| **Синтаксис** | Фигурные скобки | Круглые скобки + стрелка (опционально) |
| **Неявный параметр** | Есть (`it`) | Нет (нужно объявлять всегда) |
| **Делегирование** | Есть `this`, `owner`, `delegate` | Только область видимости внешнего класса |
| **Производительность** | Чуть медленнее (динамика) | Быстрее (ближе к нативному байткоду Java) | 
| **Сериализация** | Реализует `Serializable` | Зависит от реализации JVM |

**4. Когда что использовать?**

* **Используйте Closure**, если вам нужна динамика Groovy, работа с `delegate` (например, при написании DSL для JMeter) или краткость через `it`.
* **Используйте Лямбды**, если вы работаете с Java Streams API или если вам важна максимальная производительность в критичных узлах кода.
* **Используйте Ссылки на методы**, когда нужная логика уже описана в существующем методе — это делает код максимально чистым.

### Delegation и Scope в Closure <a id="delegation-scope"></a>

Внутри каждого замыкания (Closure) в Groovy скрыто три объекта, которые определяют, как будет искаться переменная или метод, если они не объявлены внутри самого тела замыкания.

**1. Три кита контекста: this, owner, delegate**

| Объект | На что указывает | Можно ли изменить? |
| --- | --- | --- |
| **this** | Ссылается на экземпляр **класса**, в котором было создано замыкание. | Нет |
| **owner** | Ссылается на **объект**, в котором создано замыкание (это может быть класс или другое замыкание). | Нет |
| **delegate** | По умолчанию равен `owner`. Это «запасной» объект для поиска методов и свойств. | **Да** |

**Пример для понимания иерархии:**

```groovy
class Outer {
    def nested() {
        def closure = {
            def inner = {
                println "this: " + this.class.simpleName    // Outer
                println "owner: " + owner.class.simpleName  // Outer$_nested_closure1 (замыкание выше)
                println "delegate: " + delegate.class.simpleName // То же, что owner
            }
            inner()
        }
        closure()
    }
}
new Outer().nested()

```

**2. Зачем нужен delegate?** 

Это главный инструмент для создания красивого кода. Вы можете «подменить» объект, к которому обращается замыкание.

Представь, что у тебя есть класс настроек в JMeter. Вместо того чтобы писать `config.timeout = 500`, ты хочешь написать просто `timeout = 500` внутри блока.

**Пример смены делегата:**

```groovy
def config = [timeout: 0, url: ""]
def script = {
    timeout = 1000  // Мы не пишем config.timeout
    url = "https://google.com"
}

script.delegate = config
script()

println config // [timeout:1000, url:https://google.com]

```

**3. Стратегии разрешения (Resolve Strategies)**

Когда в замыкании вызывается метод или переменная, Groovy должен решить: искать сначала в `owner` или в `delegate`? За это отвечает свойство `resolveStrategy`.

| Стратегия | Описание |
| --- | --- |
| **Closure.OWNER_FIRST** | (По умолчанию). Сначала ищем в `owner`, если там нет — идем в `delegate`. |
| **Closure.DELEGATE_FIRST** | Сначала ищем в `delegate`, если там нет — в `owner`. Идеально для DSL. |
| **Closure.OWNER_ONLY** | Игнорируем делегата полностью. |
| **Closure.DELEGATE_ONLY** | Игнорируем владельца (owner) полностью. |
| **Closure.TO_SELF** | Ищем только внутри полей самого объекта Closure (редко используется). |

**Пример с конфликтом имен:**

```groovy
class Person { String name = "Delegate Name" }
def name = "Owner Name"

def cl = { println name }

cl.delegate = new Person()
cl.resolveStrategy = Closure.DELEGATE_FIRST

cl() // Выведет: "Delegate Name", так как мы сменили приоритет

```

**4. Итог для конспекта**

* **Scope** определяется тем, где физически написан код (`this`, `owner`).
* **Delegation** — это динамическая возможность перенаправить вызовы на другой объект.
* **Использование:** В нагрузочном тестировании (JMeter) это позволяет писать лаконичные скрипты, где методы `sampler` или `vars` доступны «напрямую» в скрипте благодаря правильно настроенному делегату.

## 🟠 Управление доступом и модификаторы <a id="access-modifiers"></a>

### Public, Private, Protected <a id="public-private-protected"></a>

В Groovy есть важное правило: если вы не указали модификатор доступа, объект считается **public** (но с нюансами для полей классов).

**1. Сравнение модификаторов доступа**

| Модификатор | Видимость внутри класса | Видимость в подклассах (наследование) | Видимость в том же пакете | Видимость везде (world) | Особенности в Groovy |
| --- | --- | --- | --- | --- | --- |
| **public** | ✅ | ✅ | ✅ | ✅ | **Дефолтный** уровень для методов и классов. Писать `public` необязательно. |
| **protected** | ✅ | ✅ | ✅ | ❌ | Работает так же, как в Java: доступ для наследников и внутри пакета. |
| **private** | ✅ | ❌ | ❌ | ❌ | **Внимание:** В динамическом режиме Groovy часто позволяет обойти private (через метапрограммирование). |
| **@PackageScope** | ✅ | ❌ | ✅ | ❌ | В Groovy нет "default" (package-private) через пустое место. Нужно использовать эту аннотацию. |

**2. Разница между Полем (Field) и Свойством (Property)**

Это ключевой момент. В зависимости от того, указали ли вы модификатор, Groovy по-разному обрабатывает переменную.

| Синтаксис | Тип сущности | Что генерирует Groovy "под капотом" | Доступ |
| --- | --- | --- | --- |
| `String name` | **Property** | Генерирует private поле + **public getter и setter**. | Доступ через `obj.name` (вызывает геттер). |
| `public String name` | **Field** | Обычное публичное поле (как в Java). | Прямой доступ к полю. |
| `private String name` | **Field** | Приватное поле. Геттеры/сеттеры **не** создаются. | Доступ только внутри класса (формально). |

**3. Уровни доступа к различным элементам** 

| Элемент | Модификатор по умолчанию | Результат |
| --- | --- | --- |
| **Класс** | `public` | Класс доступен из любой точки проекта. |
| **Метод** | `public` | Метод доступен всем. `def sayHi() {}` — это публичный метод. |
| **Поле класса** | `Property` | Создаются неявные методы доступа (get/set). |
| **Локальная переменная** | `—` | Видна только в блоке кода (методе/цикле), где объявлена. |

**4. Исключения и "динамическая лазейка"**

Groovy — динамический язык. Даже если вы пометили метод как `private`, технически к нему можно обратиться через `invokedynamic` или напрямую через `object.metaClass`. Однако в нагрузочных скриптах (JMeter) лучше придерживаться стандартных правил инкапсуляции.

**Пример инкапсуляции "по-грувовому":** 

```groovy
class User {
    String name          // Public Property (авто-геттер/сеттер)
    private int age      // Private Field (геттера нет)
    protected String id  // Protected Field
    
    def getInfo() { "Name: $name, Age: $age" } // Public метод
}

def u = new User(name: "Ivan") // Работает через неявный конструктор и сеттер
u.name = "Dmitry"              // Вызывает setName("Dmitry")
// u.age = 25                  // Ошибка (если не внутри класса)

```

### Static, Final и Immutable <a id="static-final-immutable"></a>

**1. Ключевые слова: Static и Final**

Эти модификаторы работают почти так же, как в Java, но с учетом динамической природы Groovy.

| Модификатор | К чему применяется | Описание | Особенности в Groovy |
| --- | --- | --- | --- |
| **static** | Поля | Переменная принадлежит классу, а не объекту. Общая для всех экземпляров. | Можно обращаться через `Класс.поле` или `экземпляр.поле`. |
|  | Методы | Метод можно вызвать без создания объекта. | Часто используются для утилитных функций. |
|  | Инициализаторы | Блок `static { ... }` выполняется при загрузке класса. | Удобно для сложной настройки статических коллекций. |
| **final** | Переменные | Значение можно присвоить только один раз. | В Groovy `final` переменные часто используются в замыканиях. |
|  | Методы | Запрещает переопределение метода в подклассах. | Помогает зафиксировать логику базового класса. |
|  | Классы | Запрещает наследование от этого класса. | Все методы класса неявно становятся `final`. |

**2. Сравнение обычного и статического доступа**

| Признак | Обычное поле (Instance) | Статическое поле (Static) |
| --- | --- | --- |
| **Хранение** | В куче (Heap) вместе с объектом. | В Metaspace (привязано к классу). |
| **Жизненный цикл** | Живет, пока жив объект. | Живет, пока загружен класс. |
| **Доступ к `this**` | Доступен. | **Недоступен** (статический контекст не знает об объекте). |

**3. Аннотация @Immutable (Неизменяемость)**

В Groovy есть мощная аннотация `@Immutable`, которая делает класс полностью неизменяемым. Это критически важно для многопоточности и нагрузочного тестирования, так как такие объекты потокобезопасны по определению.

| Что делает @Immutable | Результат |
| --- | --- |
| **Поля становятся final** | После создания объекта значения полей нельзя изменить. |
| **Удаляются сеттеры** | Попытка вызвать `setX()` приведет к исключению. |
| **Копирование коллекций** | Если поле — `List` или `Map`, Groovy создаст их защищенную копию. |
| **Генерация методов** | Автоматически создаются корректные `equals()`, `hashCode()` и `toString()`. |
| **Конструктор** | Создается конструктор, принимающий все параметры по порядку или в виде Map. |

**Пример кода:**

```groovy
import groovy.transform.Immutable

@Immutable
class Point {
    double x, y
}

def p1 = new Point(x: 10, y: 20)
// p1.x = 30 // Ошибка: ReadOnlyPropertyException

```

**4. Итоговая таблица: когда что выбирать?**

| Ваша цель | Используйте |
| --- | --- |
| Создать константу | `static final String CONST = "value"` |
| Сделать переменную, которую нельзя менять | `final def x = 10` |
| Счетчик созданных объектов | `static int counter = 0` |
| Создать объект-значение (DTO), который нельзя повредить | `@Immutable class UserDTO { ... }` |

**5. Важный нюанс для нагрузочного тестирования (JMeter)**

При использовании `static` в скриптах JMeter (например, в JSR223 Sampler) будьте осторожны: статические переменные сохраняются между итерациями и потоками, так как Groovy скрипты часто кэшируются. Это может привести к утечкам памяти или неверным данным, если вы рассчитываете на «чистое» состояние в начале каждого теста.

### Properties и Getter/Setter <a id="properties-getter-setter"></a>

В Groovy любое поле, объявленное без модификатора доступа (или с модификатором, но без `private`), автоматически становится **свойством (Property)**.

**1. Механика свойств**

Когда вы объявляете свойство, Groovy автоматически генерирует для него "под капотом" следующие элементы:

| Компонент | Что генерируется автоматически |
| --- | --- |
| **Поле** | Закрытое поле (`private field`). |
| **Геттер** | Публичный метод `getXxx()`. |
| **Сеттер** | Публичный метод `setXxx(value)`. |

**2. Сравнение доступа к данным**

В Groovy обращение через точку (`obj.name`) — это не прямой доступ к полю, а вызов соответствующего метода.

| Действие в коде | Что происходит на самом деле | Пример |
| --- | --- | --- |
| **Чтение:** `val = user.name` | Вызов `user.getName()` | `println user.name` |
| **Запись:** `user.name = "Bob"` | Вызов `user.setName("Bob")` | `user.name = 'Alice'` |
| **Прямой доступ** | Обращение к полю в обход методов | `user.@name` (использовать с осторожностью) |

**3. Управление генерацией (Read-only и Custom)**

Вы можете легко контролировать, какие методы будут созданы.

| Задача | Решение | Результат |
| --- | --- | --- |
| **Только чтение** | Объявить поле как `final`. | Сгенерируется только `get`, сеттера не будет. |
| **Своя логика** | Написать свой `get` или `set`. | Groovy не будет генерировать стандартный, а использует ваш. |
| **Скрыть доступ** | Использовать `private`. | Поле перестает быть свойством, методы не генерируются. |

**4. Преимущества свойств в Groovy**

| Преимущество | Описание |
| --- | --- |
| **Чистый код** | Класс из 5 полей занимает 5 строк вместо 50. |
| **Uniform Access** | Код выглядит одинаково, используете вы простое поле или сложную логику в геттере. |
| **Интерполяция** | Свойства легко вставлять в GString: `"${user.name}"`. |
| **Безопасность** | Даже если вы обращаетесь к свойству как к полю, инкапсуляция сохраняется (всегда идет через метод). |

**5. Пример реализации**

```groovy
class Student {
    String name                  // Property: есть get и set
    final String university = "MIT" // Read-only Property: только get
    private int grade = 5        // Field: доступа извне нет
    
    // Кастомный сеттер для валидации
    void setName(String name) {
        println "Меняем имя на $name"
        this.name = name.toUpperCase()
    }
}

def s = new Student()
s.name = "ivan"       // Выведет лог и сохранит "IVAN"
println s.university  // MIT
// s.university = "Oxford" // Ошибка (ReadOnlyPropertyException)

```

## ⚡ Операторы и конструкции управления <a id="operators"></a>

### Арифметические и логические операторы <a id="арифметические-логические"></a>

**1. Арифметические операторы**

Groovy поддерживает все стандартные арифметические операции, но добавляет несколько удобных синтаксических упрощений.

| Оператор | Назначение | Пример | Результат | Примечание |
|:--------:|:-----------|:--------|:----------|:-----------|
| `+` | Сложение | `5 + 3` | `8` | Работает и со строками (`"Hello " + "World"`) |
| `-` | Вычитание | `10 - 4` | `6` | |
| `*` | Умножение | `6 * 7` | `42` | Работает со строками (`"Ha" * 3` → `"HaHaHa"`) |
| `/` | Деление | `10 / 3` | `3.3333333333` | Результат всегда `BigDecimal` |
| `%` | Остаток (модуль) | `10 % 3` | `1` | |
| `**` | Возведение в степень | `2 ** 3` | `8` | Специфичный для Groovy оператор |
| `++` | Инкремент | `x = 1; x++` | `2` | Постфиксная форма |
| `--` | Декремент | `y = 5; y--` | `4` | Префиксная форма работает аналогично |

**2. Операторы сравнения**

Операторы сравнения в Groovy похожи на Java, но есть ключевое отличие в работе `==`.

| Оператор | Назначение | Пример | Результат | Отличие от Java |
|:--------:|:-----------|:--------|:----------|:----------------|
| `==` | Равенство значений | `"abc" == new String("abc")` | `true` | В Java сравнивает ссылки, в Groovy — значения (вызывает `.equals()`) |
| `!=` | Неравенство | `5 != 6` | `true` | Аналогично `==` |
| `===` | Ссылочное равенство | `a === b` | `false` | Как `==` в Java (сравнивает, указывают ли на один объект) |
| `!==` | Ссылочное неравенство | `a !== b` | `true` | Проверка, что это разные объекты |
| `<` | Меньше | `3 < 5` | `true` | |
| `<=` | Меньше или равно | `5 <= 5` | `true` | |
| `>` | Больше | `7 > 2` | `true` | |
| `>=` | Больше или равно | `4 >= 6` | `false` | |
| `<=>` | Spaceship (космический корабль) | `5 <=> 3` | `1` | **Groovy-only.** Возвращает `-1` (если левый < правый), `0` (равны), `1` (левый > правый). Полезно для сортировки. |

```groovy
// Наглядная разница между == и ===
def str1 = "hello"
def str2 = new String("hello")

println str1 == str2   // true (значения равны)
println str1 === str2  // false (разные объекты)

// Пример использования <=>
def list = [3, 1, 4, 2]
list.sort { a, b -> a <=> b }  // [1, 2, 3, 4]
```

**3. Логические операторы**

Используются для работы с булевыми значениями и поддерживают *short-circuit evaluation* (ленивое вычисление).

| Оператор | Назначение | Пример | Результат | Правило |
|:--------:|:-----------|:--------|:----------|:---------|
| `&&` | Логическое **И** | `true && false` | `false` | Возвращает `true`, только если оба операнда истинны |
| `\|\|` | Логическое **ИЛИ** | `true \|\| false` | `true` | Возвращает `true`, если хотя бы один операнд истинен |
| `!` | Логическое **НЕ** | `!true` | `false` | Инвертирует значение |
| `&` | Безопасное **И** | `true & можетУпасть()` | *Ошибка* | **НЕ ленивый.** Всегда вычисляет оба операнда. Используется редко. |
| `\|` | Безопасное **ИЛИ** | `true \| можетУпасть()` | *Ошибка* | **НЕ ленивый.** Вычисляет оба операнда. |

**4. Особенности Groovy: Truthy и Falsy (правдивость)**

В Groovy (в отличие от Java) не только `boolean` может быть использован в условиях. Любое выражение имеет свою "правдивость". Это критически важно для лаконичности кода.

| Тип данных | Считается **false** | Считается **true** |
|:-----------|:---------------------|:--------------------|
| **Boolean** | `false` | `true` |
| **String** | `null` или пустая (`""`) | Любая непустая строка (даже `"false"`) |
| **Числа** | `0` | Любое ненулевое число (включая отрицательные) |
| **Коллекции** (List, Map, Set) | `null` или пустая (`[]`, `[:]`) | Любая коллекция с элементами |
| **Объект** | `null` | Любой не-null объект |
| **Iterator / Enumeration** | `null` или если `.hasNext()` → `false` | Если есть следующий элемент |
| **Matcher** (регулярки) | Если не найдено совпадений | Если найдено хотя бы одно совпадение |

**Практический пример Truthy:**
```groovy
def name = ""
def list = [1, 2, 3]

// Вместо громоздких проверок
if (name) {
    println "Имя не пустое"  // Не выполнится, т.к. строка пустая
}

if (list) {
    println "В списке есть элементы" // Выполнится, т.к. список не пуст
}

// Проверка на null и наличие элементов в одну строку
def process(items) {
    if (!items) {
        return "Коллекция пуста или null"
    }
    return items.size()
}
```

**5. Приоритет операторов (от высшего к низшему)**

1. **Постфиксные:** `++`, `--`
2. **Унарные:** `+`, `-`, `!`, `~` (побитовое НЕ)
3. **Степень:** `**`
4. **Мультипликативные:** `*`, `/`, `%`
5. **Аддитивные:** `+`, `-`
6. **Сравнения:** `<`, `<=`, `>`, `>=`, `instanceof`, `as`
7. **Равенства:** `==`, `!=`, `===`, `!==`, `<=>` (spaceship)
8. **Логическое И:** `&&`, `&`
9. **Логическое ИЛИ:** `||`, `|`
10. **Тернарный:** `? :`
11. **Присваивание:** `=`, `+=`, `-=`, `*=`, `/=`, `%=`, `**=`

### Условные конструкции: if, switch <a id="условные"></a>

**1. Конструкция if / else**

В Groovy `if` работает так же, как в Java, но использует правила **Groovy Truth** (правдивости), что позволяет писать более лаконичные условия.

| Синтаксис | Описание | Пример |
|:----------|:---------|:--------|
| `if (условие) { ... }` | Выполняется, если условие истинно | `if (user) { println user.name }` |
| `if (условие) { ... } else { ... }` | Выполняет один из блоков | `if (score >= 60) { "Pass" } else { "Fail" }` |
| `if (условие1) { ... } else if (условие2) { ... } else { ... }` | Множественный выбор | `if (x > 0) { "positive" } else if (x < 0) { "negative" } else { "zero" }` |

**Ключевая особенность**: Благодаря **Groovy Truth**, в условии можно писать:

```groovy
def list = []
def str = ""
def num = 0
def obj = null

if (list) { /* не выполнится, список пуст */ }
if (str) { /* не выполнится, строка пуста */ }
if (num) { /* не выполнится, число 0 */ }
if (obj) { /* не выполнится, объект null */ }
```

**2. Тернарный оператор и Elvis**

Groovy расширяет стандартный тернарный оператор, делая код еще короче.

| Оператор | Синтаксис | Описание | Пример |
|:---------|:----------|:---------|:--------|
| **Тернарный** | `условие ? значение1 : значение2` | Классический выбор из двух вариантов | `def status = age >= 18 ? "adult" : "minor"` |
| **Elvis** | `выражение ?: значениеПоУмолчанию` | Если слева **truthy** — возвращает его, иначе — значение справа | `def name = inputName ?: "Guest"` |

**Разница между тернарным и Elvis:**

```groovy
// Тернарный оператор (нужно явно повторять переменную)
def displayName = name != null ? name : "Anonymous"

// Elvis (короткое замыкание на null/пустоте)
def displayName = name ?: "Anonymous"  
// Работает, потому что null и пустая строка — "falsy"

// Другой пример с дефолтными значениями
def timeout = config.timeout ?: 3000  // Если не задано, то 3000 мс
```

**3. Конструкция switch (мощь Groovy)**

`switch` в Groovy гораздо мощнее, чем в Java. Он может работать с любым типом объекта, а в `case` можно использовать не только константы, но и диапазоны, коллекции, замыкания и регулярные выражения.

| Тип `case` | Что проверяет | Пример |
|:-----------|:--------------|:--------|
| **Значение** | Равенство (`==`) | `case 42:` |
| **Диапазон (Range)** | Вхождение в диапазон (`in`) | `case 18..65:` |
| **Коллекция (List/Set)** | Вхождение в коллекцию (`in`) | `case ['Y', 'N']:` |
| **Тип класса** | Проверка типа (`instanceof`) | `case Integer:` |
| **Регулярное выражение** | Совпадение с паттерном (`==~`) | `case ~/\d+/:` |
| **Замыкание (Closure)** | Вызов замыкания (truthy результата) | `case { it % 2 == 0 }:` |

**Синтаксис и пример:**

```groovy
def evaluate(value) {
    switch (value) {
        case 0:
            return "Ноль"
            
        case 1..10:
            return "Маленькое число"
            
        case 11..100:
            return "Среднее число"
            
        case [101, 200, 500]:  // Конкретные значения в списке
            return "Особый код"
            
        case Integer:
            return "Целое число больше 100"
            
        case ~/^[A-Z]+$/:  // Регулярка: только заглавные буквы
            return "Только заглавные"
            
        case { it instanceof String && it.length() > 10 }:
            return "Длинная строка"
            
        case String:
            return "Обычная строка"
            
        default:
            return "Неизвестный тип: ${value.class.name}"
    }
}

// Проверяем
println evaluate(5)        // Маленькое число
println evaluate(50)       // Среднее число
println evaluate(200)      // Особый код
println evaluate(1000)     // Целое число больше 100
println evaluate("HELLO")  // Только заглавные
println evaluate("hello")  // Обычная строка
println evaluate([1,2,3])  // Неизвестный тип: ArrayList
```

**4. Сравнение if и switch**

| Критерий | `if` | `switch` |
|:---------|:-----|:---------|
| **Гибкость** | Любое булево выражение | Проверка одного выражения на разные условия |
| **Читаемость** | Хорошо для 1-2 условий | Отлично для множества вариантов |
| **Производительность** | Обычно быстрее для простых проверок | Оптимизируется JVM для констант |
| **Типы условий** | Только `boolean` | Любые типы + мощные `case` |
| **Сложные проверки** | Любая логика | Только через замыкания в `case` |

**5. Идиоматический Groovy: switch с замыканиями**

```groovy
def processPayment(method) {
    switch (method.type) {
        case { it.isCreditCard() }:
            return handleCreditCard(method)
            
        case { it.isPayPal() }:
            return handlePayPal(method)
            
        case { it.amount > 10000 }:
            logLargePayment(method)
            // Проваливаемся дальше (fall through)
            
        case { it.isCash() }:
            return handleCash(method)
            
        default:
            throw new IllegalArgumentException("Неизвестный метод оплаты")
    }
}
```

**6. Важное замечание о fall-through**

В Groovy, как и в Java, после выполнения `case` без `break` или `return` происходит "проваливание" (fall-through) к следующему `case`. Но Groovy позволяет использовать `return` внутри `case`, что часто удобнее.

```groovy
// Без break - осторожно!
def test(x) {
    switch (x) {
        case 1:
            println "Один"
            // Провалится к следующему!
        case 2:
            println "Два"
            break
        default:
            println "Другое"
    }
}
test(1) // Выведет "Один" и "Два" (провал)
```

### Циклы: for, while, each <a id="циклы"></a>

**1. Традиционные циклы (Java-стиль)**

Groovy поддерживает классические циклы `while` и `for` в стиле Java, но добавляет свои мощные итерационные возможности.

| Тип цикла | Синтаксис | Когда использовать | Пример |
|:----------|:----------|:-------------------|:--------|
| **while** | `while (условие) { ... }` | Когда количество итераций неизвестно заранее | `while (queue.hasNext()) { ... }` |
| **do-while** | `do { ... } while (условие)` | Когда тело должно выполниться хотя бы раз | `do { x = readInput() } while (!x.valid)` |
| **classic for** | `for (i=0; i<10; i++) { ... }` | Когда нужен индекс и точное количество | `for (int i=0; i<list.size(); i++) { ... }` |

```groovy
// while
def count = 0
while (count < 3) {
    println "Итерация: ${count++}"
}

// do-while
def input = ""
do {
    input = getUserInput()  // выполнится хотя бы раз
} while (!input.isValid())

// Классический for
def list = ['a', 'b', 'c']
for (int i = 0; i < list.size(); i++) {
    println "list[$i] = ${list[i]}"
}
```

**2. For-each по коллекциям (Java-стиль)**

Groovy поддерживает Java-синтаксис `for (element : collection)`, но обычно предпочитают более лаконичные методы `each`.

| Синтаксис | Описание | Пример |
|:----------|:---------|:--------|
| `for (element in collection) { ... }` | Итерация по элементам | `for (item in list) { println item }` |
| `for (key in map.keySet()) { ... }` | По ключам Map | `for (k in map.keySet()) { println k }` |
| `for (entry in map) { ... }` | По записям Map | `for (kv in map) { println kv.key + ":" + kv.value }` |

```groovy
def names = ['Alice', 'Bob', 'Charlie']
def ages = [Alice: 25, Bob: 30, Charlie: 35]

// По элементам списка
for (name in names) {
    println "Hello, $name"
}

// По записям Map
for (entry in ages) {
    println "${entry.key} is ${entry.value} years old"
}
```

**3. Groovy-стиль: each и eachWithIndex**

В Groovy принято использовать методы-итераторы, передавая им замыкания. Это лаконичнее и функциональнее.

| Метод | Назначение | Параметры замыкания | Пример |
|:------|:-----------|:-------------------|:--------|
| **`each`** | Итерация по элементам | `{ элемент → }` | `list.each { println it }` |
| **`eachWithIndex`** | Итерация с индексом | `{ элемент, индекс → }` | `list.eachWithIndex { item, i → println "$i: $item" }` |
| **`reverseEach`** | Обратная итерация | `{ элемент → }` | `list.reverseEach { println it }` |

```groovy
def fruits = ['apple', 'banana', 'orange']

// each (с неявным it)
fruits.each { println "I like $it" }

// each (с явным параметром)
fruits.each { fruit -> println "I like $fruit" }

// eachWithIndex
fruits.eachWithIndex { fruit, i ->
    println "$i. $fruit"
}

// Для Map
def capitals = [France: 'Paris', Italy: 'Rome', Spain: 'Madrid']
capitals.each { country, city ->
    println "The capital of $country is $city"
}
```

**4. Итерация по диапазонам (Range)**

Диапазоны — это "гражданские" объекты в Groovy, которые можно итерировать напрямую.

| Способ | Синтаксис | Пример |
|:-------|:----------|:--------|
| **for-in с Range** | `for (i in 1..5) { ... }` | `for (i in 1..3) { println "Раз: $i" }` |
| **each с Range** | `(1..5).each { ... }` | `(1..5).each { println "Квадрат $it = ${it*it}" }` |
| **step** | `range.step(2) { ... }` | `(0..10).step(2) { println "Четное: $it" }` |

```groovy
// Классический for с шагом
for (i in 0..10 step 2) {
    println i  // 0,2,4,6,8,10
}

// each с диапазоном символов
('a'..'f').each { println "Буква: $it" }

// step для сложной логики
(1..100).step(10) { num ->
    println "Десяток: $num"  // 1,11,21,...,91
}
```

**5. Управление циклами: break, continue, return**

| Оператор | Действие | Применимость | Пример |
|:---------|:---------|:-------------|:--------|
| **`break`** | Полный выход из цикла | Любые циклы | `for (x in list) { if (x == 'stop') break }` |
| **`continue`** | Переход к следующей итерации | Любые циклы | `for (x in list) { if (x == null) continue }` |
| **`return`** | Выход из метода (не только цикла) | В методах | `def find(list) { list.each { if (it) return it } }` |

**Важное отличие:** В замыканиях `return` выходит **только из замыкания**, а не из внешнего метода. Для выхода из внешнего метода нужно использовать `break` с меткой.

```groovy
// break и continue работают как обычно
def numbers = [1,2,3,4,5,6]
for (num in numbers) {
    if (num % 2 == 0) continue    // пропускаем четные
    if (num > 4) break             // останавливаем после 5
    println num                    // выведет: 1,3
}

// Метки для вложенных циклов
outer: for (i in 1..3) {
    for (j in 1..3) {
        if (i * j > 4) break outer
        println "$i * $j = ${i * j}"
    }
}
// Выведет: 1*1=1, 1*2=2, 1*3=3, 2*1=2, 2*2=4 и останов
```

**6. Сравнение производительности и стиля**

| Метод | Производительность | Читаемость | Когда использовать |
|:------|:-------------------|:-----------|:-------------------|
| **classic for** | Высокая | Средняя | Точные индексы, старые привычки |
| **for-in** | Высокая | Хорошая | Простая итерация по коллекции |
| **each** | Средняя | Отличная | Идиоматичный Groovy-код, цепочки методов |
| **eachWithIndex** | Средняя | Хорошая | Когда нужен и индекс, и элемент |

**7. Идиоматические примеры для JMeter/нагрузочного тестирования**

```groovy
// Генерация тестовых данных
def users = []
(1..100).each { id ->
    users << [
        id: id,
        name: "User_${id}",
        email: "user${id}@test.com"
    ]
}

// Парсинг CSV-данных (если есть список строк)
def csvLines = ['id,name', '1,Alice', '2,Bob']
csvLines.each { line ->
    if (line.startsWith('id')) return  // пропускаем заголовок
    def parts = line.split(',')
    println "User ${parts[0]}: ${parts[1]}"
}

// Пакетная обработка с прогрессом
def items = (1..1000).toList()
items.eachWithIndex { item, idx ->
    if (idx % 100 == 0) {
        println "Обработано $idx из ${items.size()}"
    }
    // ... обработка item
}
```

### Elvis, Safe Navigation и Spread-операторы <a id="elvis-safe"></a>

**1. Оператор безопасной навигации (Safe Navigation) `?.`**

Этот оператор спасает от `NullPointerException` при обращении к методам или свойствам потенциально `null` объектов. Если объект слева — `null`, оператор просто возвращает `null`, не вызывая метод/свойство.

| Синтаксис | Что делает | Эквивалентный Java-код |
|:----------|:-----------|:----------------------|
| `user?.name` | Вернет `user.name`, если `user != null`, иначе `null` | `user != null ? user.name : null` |
| `list?.size()` | Вернет размер списка или `null` | `list != null ? list.size() : null` |
| `obj?.method()?.property` | Безопасная цепочка вызовов | Цепочка вложенных `if (obj != null)` |

```groovy
def user = null
def city = user?.address?.city  // Безопасно: вернет null, а не NPE

def users = ['Alice', 'Bob']
println users?.size()  // 2

users = null
println users?.size()  // null (не 0, а именно null!)

// Смешиваем с Elvis для дефолтных значений
def timeout = config?.timeout ?: 5000  // Если config или timeout null → 5000
```

**2. Оператор Elvis (Elvis Operator) `?:`**

Назван так, потому что похож на прическу Элвиса Пресли. Используется для предоставления значения по умолчанию, когда выражение слева "falsy" (null, пустая строка, 0 и т.д. по правилам Groovy Truth).

| Синтаксис | Что делает | Пример |
|:----------|:-----------|:--------|
| `x ?: y` | Если `x` **truthy** → возвращает `x`, иначе возвращает `y` | `def name = input ?: "Guest"` |
| `(x ?: y).method()` | Можно использовать в цепочках | `(params.id ?: "0").toInteger()` |

```groovy
// Классический пример: дефолтное имя
def username = session.username ?: "Anonymous"

// С числами (0 считается falsy!)
def count = getCount() ?: 100  // если вернулся 0, возьмем 100

// С коллекциями
def activeUsers = cachedUsers ?: fetchUsersFromDB()

// Цепочка Elvis'ов
def port = systemPort ?: configPort ?: defaultPort ?: 8080

// Внимание: пустая строка тоже falsy!
def text = "" ?: "default"  // вернет "default"
```

**3. Оператор распространения (Spread Operator) `*.`**

Позволяет вызвать метод или получить свойство у **каждого элемента** коллекции, собирая результаты в новую коллекцию.

| Синтаксис | Что делает | Результат | Пример |
|:----------|:-----------|:-----------|:--------|
| `list*.property` | Взять свойство у каждого элемента | Новый `List` | `users*.name` → список имен |
| `list*.method()` | Вызвать метод у каждого | Новый `List` | `strings*.toUpperCase()` |
| `list*.method().*.property` | Цепочка spread-операторов | Цепочка трансформаций | `users*.address*.city` |

```groovy
class Person {
    String name
    int age
    Address address
}

class Address {
    String city
}

def people = [
    new Person(name: 'Alice', age: 25, address: new Address(city: 'NYC')),
    new Person(name: 'Bob', age: 30, address: new Address(city: 'LA')),
    new Person(name: 'Charlie', age: 35)
]

// Получаем все имена
def names = people*.name  // ['Alice', 'Bob', 'Charlie']

// Все возраста
def ages = people*.age     // [25, 30, 35]

// Цепочка: города (у Charlie города нет → null)
def cities = people*.address*.city  // ['NYC', 'LA', null]

// Вызов методов
def upperNames = people*.name*.toUpperCase()  // ['ALICE', 'BOB', 'CHARLIE']

// Смешиваем с безопасной навигацией
def safeCities = people*.address?.city  // безопасно, но spread уже дает null для отсутствующих
```

**4. Оператор распространения для Map (Spread Map) `*.` и `*:`**

В Groovy можно "разворачивать" одну Map внутрь другой.

| Синтаксис | Что делает | Пример |
|:----------|:-----------|:--------|
| `[*:map1, *:map2]` | Слияние двух Map | `def merged = [*:defaults, *:overrides]` |
| `[key: value, *:otherMap]` | Добавление ключей из другой Map | `[name: 'test', *:configMap]` |

```groovy
def defaults = [timeout: 1000, retries: 3]
def overrides = [timeout: 5000, debug: true]

// Слияние с приоритетом overrides (позже перезаписывает)
def config = [*:defaults, *:overrides]  // [timeout:5000, retries:3, debug:true]

// Добавление в существующую Map
def base = [id: 123]
def extended = [*:base, type: 'user', active: true]  // [id:123, type:'user', active:true]
```

**5. Оператор распространения для аргументов метода (Spread Arguments) `*`**

Позволяет "развернуть" список в аргументы метода, когда метод ожидает переменное число аргументов.

| Синтаксис | Что делает | Пример |
|:----------|:-----------|:--------|
| `method(*list)` | Передать элементы списка как отдельные аргументы | `def nums = [1,2,3]; sum(*nums)` |
| `method(*array)` | То же для массивов | `int[] arr = [1,2,3]; sum(*arr)` |

```groovy
def sum(int a, int b, int c) { a + b + c }

def numbers = [10, 20, 30]
println sum(*numbers)  // 60 (эквивалент sum(10,20,30))

def range = 1..3
println sum(*range)    // 6 (1+2+3)

// Смешанные аргументы
def format(String template, String... args) { template.format(args) }
def params = ['Alice', 25]
println format('Name: %s, Age: %d', *params)  // Name: Alice, Age: 25
```

**6. Сравнительная таблица операторов**

| Оператор | Название | Назначение | Возврат при `null` |
|:--------:|:---------|:-----------|:-------------------|
| `?.` | Safe Navigation | Безопасный вызов метода/свойства | `null` |
| `?:` | Elvis | Значение по умолчанию | Значение справа (если слева falsy) |
| `*.` | Spread | Вызов метода для всех элементов | `null` (если коллекция `null`) |
| `*:` | Spread Map | Слияние Map | Ошибка (если Map `null`) |
| `*` | Spread Arg | Разворачивание списка в аргументы | Ошибка (если список `null`) |

**7. Комбо-примеры для нагрузочного тестирования (JMeter)**

```groovy
// Безопасное чтение переменных JMeter
def userId = vars?.get('userId') ?: "default_${UUID.randomUUID()}"
def timeout = props?.get('timeout')?.toInteger() ?: 5000

// Обработка списка сэмплеров
def samples = ctx?.getPreviousResult()?.getSamples() ?: []
def responseTimes = samples*.getTime()  // список времен ответа
def avgTime = responseTimes.sum() ?: 0 / (responseTimes.size() ?: 1)

// Генерация JSON из данных
def userData = [
    id: userId,
    tags: ['premium', 'active'],
    metadata: [*:defaultMetadata, *:customMetadata]  // слияние метаданных
]

// Безопасная цепочка
def city = vars?.getObject('user')?.address?.city ?: 'Unknown'
```

**8. Важные нюансы**

| Оператор | Подводный камень | Пример проблемы |
|:---------|:-----------------|:----------------|
| `?.` | Возвращает `null`, а не пустую коллекцию | `list?.size()` может вернуть `null`, а не 0 |
| `?:` | 0 и пустая строка считаются falsy | `count ?: 100` заменит 0 на 100 (может быть неожиданно) |
| `*.` | Создает новый список каждый раз | В больших циклах может создавать мусор для GC |
| `*` | Не работает с `null` | `sum(*null)` бросит исключение |

```groovy
// Как обойти нюансы
def safeSize = list?.size() ?: 0  // теперь всегда число
def safeCount = (count != null) ? count : 100  // если 0 должен остаться 0
def safeSpread = list ? list*.property : []    // защита от null
```

## ❌ Исключения и обработка ошибок <a id="exceptions"></a>

### Типы исключений в Groovy <a id="типы-исключений"></a>

**1. Иерархия исключений**

Groovy полностью наследует иерархию исключений Java, но добавляет несколько своих типов и важное отличие: **проверяемые исключения не требуют обязательной обработки**.

| Уровень | Класс исключения | Описание | Проверяемое? |
|:-------:|:-----------------|:---------|:-------------|
| **Верхний** | `Throwable` | Все, что можно "бросить" | — |
| ├─ **Ошибки** | `Error` | Критические проблемы JVM (обычно не ловим) | Нет |
| │ ├─ | `OutOfMemoryError` | Закончилась память | Нет |
| │ ├─ | `StackOverflowError` | Слишком глубокая рекурсия | Нет |
| │ └─ | `NoClassDefFoundError` | Класс не найден при загрузке | Нет |
| └─ **Исключения** | `Exception` | Проблемы, которые можно обработать | — |
| ├─ **Runtime** | `RuntimeException` | Ошибки времени выполнения | **Нет** |
| │ ├─ | `NullPointerException` | Обращение к null | Нет |
| │ ├─ | `IllegalArgumentException` | Неверный аргумент | Нет |
| │ ├─ | `IndexOutOfBoundsException` | Выход за границы | Нет |
| │ ├─ | `ClassCastException` | Ошибка приведения типа | Нет |
| │ └─ | `ArithmeticException` | Арифметическая ошибка (/0) | Нет |
| └─ **Проверяемые** | `Exception` (кроме RuntimeException) | Должны быть обработаны | **Да (в Java)** |

**2. Ключевое отличие Groovy от Java**

| Характеристика | Java | Groovy |
|:---------------|:-----|:--------|
| **Проверяемые исключения** | Требуют `try-catch` или `throws` | **Не требуют** обработки |
| **Код с IOException** | `try { ... } catch (IOException e) { ... }` | Можно просто написать код, компилятор не заставит ловить |
| **Философия** | Безопасность на этапе компиляции | Удобство и краткость |

```groovy
// В Java это не скомпилируется без try-catch
// В Groovy - работает!
def text = new File('/tmp/test.txt').text  // если файла нет - будет исключение в рантайме

// Но ловить все равно нужно, если хотим обработать
try {
    def data = new URL('http://example.com').text
} catch (IOException e) {
    println "Ошибка сети: ${e.message}"
}
```

**3. Специфичные для Groovy исключения**

Groovy добавляет свои типы исключений, которые можно встретить при использовании динамических возможностей языка.

| Исключение | Когда возникает | Пример |
|:-----------|:-----------------|:--------|
| `MissingMethodException` | Вызов несуществующего метода | `"string".someUnknownMethod()` |
| `MissingPropertyException` | Доступ к несуществующему свойству | `def map = [:]; map.unknownKey` |
| `GroovyCastException` | Ошибка приведения типов | `(Integer) "not a number"` |
| `MultipleCompilationErrorsException` | Ошибки компиляции скрипта | Синтаксические ошибки в скрипте |
| `ClosureCreationException` | Ошибка создания замыкания | Неверный синтаксис замыкания |
| `ReadOnlyPropertyException` | Попытка изменить read-only свойство | Изменение свойства `@Immutable` класса |

```groovy
// Примеры специфичных исключений
def map = [name: 'Alice']

try {
    println map.age  // нет такого ключа
} catch (MissingPropertyException e) {
    println "Свойство не найдено: ${e.property}"
}

try {
    def num = (Integer) "abc"
} catch (GroovyCastException e) {
    println "Нельзя преобразовать строку в число"
}
```

**4. Исключения в динамическом коде**

Благодаря метапрограммированию, в Groovy могут возникать исключения, связанные с динамическим поведением.

| Исключение | Описание | Типичный сценарий |
|:-----------|:---------|:-------------------|
| `GroovyRuntimeException` | Общее исключение времени выполнения | Проблемы с метапрограммированием |
| `IllegalAccessError` | Попытка доступа к приватному методу через метапрограммирование | `metaClass.getMetaMethod('privateMethod').invoke(obj)` |
| `StackOverflowError` | Бесконечная рекурсия в замыканиях | Замыкание, вызывающее само себя без условия выхода |

**5. Сравнение обработки исключений в Java и Groovy**

| Ситуация | Java | Groovy |
|:---------|:-----|:--------|
| **Чтение файла** | `try-catch-finally` обязателен | Можно без `try`, исключение упадет выше |
| **Парсинг числа** | `try { Integer.parseInt(s) } catch (NumberFormatException e)` | То же самое, но `NumberFormatException` непроверяемое |
| **Работа с сетью** | Обязательно ловить `IOException` | Можно не ловить (но лучше ловить) |
| **Свой синтаксис** | Только стандартный | Можно использовать `withStream { ... }` (авто-закрытие) |

**6. Практические рекомендации для JMeter/Groovy скриптов**

| Тип исключения | Как обрабатывать | Почему важно |
|:---------------|:-----------------|:-------------|
| `MissingPropertyException` | Проверять `vars.get("key") != null` | Переменные JMeter могут отсутствовать |
| `NumberFormatException` | Использовать `?.toInteger() ?: defaultValue` | Парсинг параметров из пользователей |
| `SocketTimeoutException` | Ловить и логировать, retry | Сеть может быть нестабильной |
| `OutOfMemoryError` | **Не ловить!** | Скрипт должен упасть, чтобы обнаружить утечку |
| `Exception` (общее) | Ловить в корне скрипта и логировать | Чтобы тест не падал молча |

```groovy
// Пример безопасного кода для JMeter
try {
    def timeout = vars.get("timeout")?.toInteger() ?: 5000
    def response = httpClient.get("https://api.example.com", timeout)
    
    if (response.status != 200) {
        throw new RuntimeException("HTTP ${response.status}: ${response.body}")
    }
    
    SampleResult.setSuccessful(true)
    SampleResult.setResponseData(response.body.bytes)
    
} catch (SocketTimeoutException e) {
    SampleResult.setSuccessful(false)
    SampleResult.setResponseMessage("Timeout: ${e.message}")
    log.warn("Таймаут запроса", e)
    
} catch (NumberFormatException e) {
    SampleResult.setSuccessful(false)
    SampleResult.setResponseMessage("Неверный формат timeout")
    
} catch (Exception e) {
    SampleResult.setSuccessful(false)
    SampleResult.setResponseMessage("Ошибка: ${e.getClass().simpleName}")
    log.error("Неожиданная ошибка в скрипте", e)
}
```

**7. Таблица: Когда какое исключение ловить**

| Ситуация | Какое исключение может возникнуть | Нужно ли ловить |
|:---------|:-----------------------------------|:----------------|
| Деление на ноль | `ArithmeticException` | Да, если данные приходят от пользователя |
| Парсинг числа | `NumberFormatException` | **Обязательно** |
| Доступ к null | `NullPointerException` | Использовать `?.` вместо `try-catch` |
| Чтение файла | `FileNotFoundException`, `IOException` | Да, если файл может отсутствовать |
| Запись в файл | `IOException`, `SecurityException` | Да |
| HTTP запрос | `SocketTimeoutException`, `UnknownHostException` | **Обязательно** в нагрузочном тесте |
| Преобразование типов | `GroovyCastException` | Проверять типы до преобразования |
| Рефлексия/метапрограммирование | `MissingMethodException` | Проверять `respondsTo()` или `hasProperty()` |

**8. Итог: Особенности исключений в Groovy**

| Особенность | Описание | Влияние на код |
|:------------|:---------|:---------------|
| **Проверяемые ≠ обязательные** | Можно не ловить IOException | Меньше бойлерплейта |
| **Динамические исключения** | Появляются из-за метапрограммирования | Нужно знать API, которое используешь |
| **Truthy в условиях** | Исключение - это объект, он truthy | `if (e)` всегда true, нужно проверять тип |
| **Множественный catch** | Как в Java 7+ | `catch (IOException \| SQLException e)` |
| **try-with-resources** | Есть через `.withCloseable()` | `new FileInputStream(f).withCloseable { ... }` |

### try-catch-finally <a id="try-catch-finally"></a>

Обработка исключений в Groovy очень похожа на Java, но с некоторыми синтаксическими упрощениями и особенностями, связанными с динамической природой языка. Основная конструкция — `try-catch-finally`.

**1. Базовая структура**

Конструкция состоит из трех блоков, два из которых (`catch` и `finally`) являются опциональными, но хотя бы один из них должен присутствовать после `try`.

| Блок | Обязательность | Назначение |
|:-----|:----------------|:-----------|
| `try` | **Да** | Содержит код, который может выбросить исключение. |
| `catch` | Нет (но нужен либо `catch`, либо `finally`) | Ловит и обрабатывает исключение определенного типа. |
| `finally` | Нет (но нужен либо `catch`, либо `finally`) | Выполняется **всегда** после `try`/`catch`, используется для освобождения ресурсов. |

**2. Синтаксис и пример**

```groovy
try {
    // Код, который может выбросить исключение
    def result = 10 / 0  // Арифметическое исключение
    println "Результат: $result" // Эта строка не выполнится
} catch (ArithmeticException e) {
    // Обработка конкретного исключения
    println "Ошибка: деление на ноль! (${e.message})"
} catch (Exception e) {
    // Обработка любого другого исключения
    println "Произошла ошибка: ${e.getClass().simpleName} - ${e.message}"
} finally {
    // Этот блок выполнится ВСЕГДА
    println "Блок finally: очистка ресурсов"
}

// Вывод:
// Ошибка: деление на ноль! (/ by zero)
// Блок finally: очистка ресурсов
```

**3. Множественные блоки `catch`**

Как и в Java, вы можете использовать несколько блоков `catch` для обработки разных типов исключений. Важно помнить о порядке: сначала должны идти более конкретные исключения, затем — более общие.

```groovy
try {
    def numbers = [1, 2, 3]
    def value = numbers[5] // Выбросит IndexOutOfBoundsException
    def text = null
    println text.length()  // Этот код уже не выполнится
} catch (IndexOutOfBoundsException e) {
    println "Ошибка индекса: ${e.message}"
} catch (NullPointerException e) {
    println "Ошибка: обращение к null"
} catch (RuntimeException e) {
    // Сюда попадут все остальные RuntimeException
    println "Ошибка выполнения: ${e.message}"
} catch (Exception e) {
    // Сюда попадут все проверяемые исключения
    println "Общая ошибка: ${e.message}"
}
```

**4. Особенности `catch` в Groovy (один блок для нескольких исключений)**

Начиная с Java 7 и в Groovy (который часто идет в ногу с современным Java-синтаксисом), можно перехватывать несколько типов исключений в одном блоке `catch`, используя оператор `|` (pipe).

```groovy
import java.sql.SQLException
import java.net.ConnectException

try {
    // Код, который может выкинуть разные исключения
    def connection = new URL("http://unknown.url").openConnection()
    def sql = Class.forName("com.mysql.jdbc.Driver")
} catch (ConnectException | SQLException | ClassNotFoundException e) {
    // Единая обработка для ошибок сети, БД или отсутствия драйвера
    println "Проблема с подключением или загрузкой: ${e.message}"
    log.error("Сетевая или БД ошибка", e)
}
```

**5. Блок `finally` — гарантия выполнения**

Блок `finally` выполняется всегда, независимо от того, было выброшено исключение или нет. Это идеальное место для закрытия файлов, потоков, соединений с БД или освобождения других ресурсов.

```groovy
def reader
try {
    reader = new FileReader("/tmp/data.txt")
    def line = reader.readLine()
    println "Первая строка: $line"
    // Если здесь произойдет исключение (например, при чтении),
    // reader все равно закроется в finally
} catch (FileNotFoundException e) {
    println "Файл не найден: ${e.message}"
} finally {
    // Этот блок выполнится и после успешного try, и после catch
    if (reader != null) {
        try {
            reader.close()
            println "Ресурс закрыт"
        } catch (IOException e) {
            println "Ошибка при закрытии: ${e.message}"
        }
    }
}
```

**6. Упрощенная работа с ресурсами: `.withCloseable()` и `.withStream()`**

Groovy предлагает более элегантный способ работы с ресурсами, которые нужно закрывать, — метод `withCloseable`. Он автоматически закрывает ресурс после выполнения замыкания, избавляя от необходимости писать `try-catch-finally` вручную.

| Метод | Для кого | Пример |
|:------|:---------|:--------|
| `withCloseable` | Любые классы, реализующие `Closeable` | `new FileInputStream("file.txt").withCloseable { stream -> ... }` |
| `withStream` / `withReader` / `withWriter` | Специализированные методы для I/O | `new File("file.txt").withReader { reader -> ... }` |

```groovy
// Вместо громоздкого try-catch-finally
new File("/tmp/data.txt").withReader { reader ->
    // reader автоматически закроется после выхода из замыкания
    def line = reader.readLine()
    println "Первая строка: $line"
    // Если здесь исключение, reader все равно закроется
} // catch не нужен, но если нужно обработать исключение, то try-catch все равно понадобится снаружи

// Для InputStream
def data = new FileInputStream("/tmp/file.bin").withCloseable { stream ->
    // работаем с stream, он закроется автоматически
    return stream.readAllBytes()
}
```

**7. `try-catch` как выражение (Groovy 3+)**

Начиная с Groovy 3.0, конструкция `try-catch` может возвращать значение, что делает ее похожей на тернарный оператор.

```groovy
def result = try {
    Integer.parseInt("123")
} catch (NumberFormatException e) {
    -1  // Значение по умолчанию при ошибке
}

println result // 123

def result2 = try {
    Integer.parseInt("abc")
} catch (NumberFormatException e) {
    -1
}

println result2 // -1
```

**8. Сравнение с Java**

| Критерий | Java | Groovy |
|:---------|:-----|:--------|
| **Проверяемые исключения** | Требуют обработки или `throws` | Можно игнорировать, но лучше обрабатывать |
| **Multi-catch** | Да (с Java 7) | Да |
| **try-catch как выражение** | Нет | Да (с Groovy 3) |
| **try-with-resources** | Да (с Java 7) | Да (есть `withCloseable` и I/O-методы) |
| **Обязательность `catch` или `finally`** | Да | Да |

**9. Идиоматичный Groovy-код для JMeter**

В скриптах для нагрузочного тестирования обработка исключений критически важна, чтобы один упавший запрос не остановил весь тест.

```groovy
try {
    // 1. Получаем параметры
    def userId = vars.get("userId")?.toInteger() ?: 1

    // 2. Выполняем HTTP-запрос (например, через HTTP Request sampler)
    //    В JSR223 Sampler результат автоматически попадает в SampleResult
    def response = sampler.sample() // упрощенно

    // 3. Проверяем результат
    if (SampleResult.isSuccessful()) {
        SampleResult.setResponseMessage("OK")
    } else {
        // Логируем неуспешный ответ, но не бросаем исключение
        log.warn("Request failed with code: ${SampleResult.getResponseCode()}")
    }
} catch (NumberFormatException e) {
    // Ошибка в тестовых данных - помечаем сэмпл как неуспешный, но продолжаем тест
    SampleResult.setSuccessful(false)
    SampleResult.setResponseMessage("Invalid userId format")
    log.error("Data error: ", e)
} catch (SocketTimeoutException e) {
    // Таймаут - частая проблема в нагрузке
    SampleResult.setSuccessful(false)
    SampleResult.setResponseMessage("Timeout")
    log.warn("Timeout for userId: ${vars.get('userId')}")
} catch (Exception e) {
    // Любая другая непредвиденная ошибка
    SampleResult.setSuccessful(false)
    SampleResult.setResponseMessage("Script error: ${e.getClass().simpleName}")
    log.error("Unexpected error in script", e)
} finally {
    // Здесь можно, например, инкрементировать счетчик итераций
    println "Обработка пользователя ${vars.get('userId')} завершена"
}
```

**Резюме:** `try-catch-finally` в Groovy работает привычно для Java-разработчика, но добавляет удобства: multi-catch, возможность использовать конструкцию как выражение и встроенные методы для автоматического закрытия ресурсов, что делает код чище и безопаснее.

### Throw и custom exceptions <a id="throw-custom"></a>

**1. Оператор `throw`: Генерация исключений**

Оператор `throw` используется для явного выброса исключения в программе. В Groovy можно выбросить как стандартные исключения Java, так и свои собственные. Важное отличие от Java: `throw` в Groovy — это не оператор, а **выражение**, что позволяет использовать его в более гибких конструкциях.

| Характеристика | Описание |
|:---|:---|
| **Синтаксис** | `throw new ExceptionClass("Сообщение об ошибке")` |
| **Тип выражения** | Является выражением (может стоять справа от присваивания, в тернарном операторе) |
| **Возвращаемое значение** | Никогда не возвращает управление (имеет тип `Nothing`) |
| **Что можно бросить** | Любой объект, являющийся подклассом `java.lang.Throwable` |

**Особенности `throw` как выражения:**

Благодаря тому, что `throw` — это выражение, можно писать лаконичный код:

```groovy
// В тернарном операторе
def result = value != null ? value : throw new IllegalArgumentException("value is null")

// В лямбдах и замыканиях
def riskyOperation = { input ->
    input ?: throw new RuntimeException("No input provided")
}
```

**2. Создание пользовательских исключений (Custom Exceptions)**

Groovy наследует объектную модель Java, поэтому создание своих классов исключений происходит стандартным способом: наследованием от `Exception` (проверяемое) или `RuntimeException` (непроверяемое).

| Тип исключения | Базовый класс | Когда использовать | Требование к обработке |
|:---|:---|:---|:---|
| **Проверяемое (Checked)** | `Exception` (или его подклассы, не наследующие `RuntimeException`) | Ошибка, от которой клиентский код **обязан** оправиться (например, ошибка бизнес-логики, которую можно обработать). | В Java — обязательно. В Groovy — **не обязательно**, но рекомендуется для ясности. |
| **Непроверяемое (Unchecked)** | `RuntimeException` | Ошибка, возникшая из-за состояния программы, которую клиент обычно не может обработать (например, неверный аргумент, нарушение внутренней логики). | Не требуется ни в Java, ни в Groovy. |

**3. Сравнение способов создания кастомных исключений**

Groovy предлагает несколько способов объявления своих исключений, от классического Java-стиля до более лаконичных.

| Способ | Синтаксис | Преимущества | Недостатки |
|:---|:---|:---|:---|
| **Классический класс** | `class MyException extends RuntimeException {`<br>`    MyException(String msg) { super(msg) }`<br>`}` | Полный контроль над конструкторами и логикой. | Многословно. |
| **Компактный класс** | `class MyException extends RuntimeException {}` | Минимально. Базовый конструктор с сообщением доступен. | Нет кастомных конструкторов. |
| **Аннотация `@InheritConstructors`** | `@InheritConstructors`<br>`class MyException extends RuntimeException {}` | **Groovy-стиль.** Автоматически наследует все конструкторы родителя. | Требует импорта аннотации. |
| **Анонимный класс (редко)** | `def ex = new RuntimeException("msg") {}` | Быстро, "на лету". | Нельзя переиспользовать; нет имени типа. |

**4. Сравнение: Пользовательские исключения в Java vs Groovy**

| Критерий | Java | Groovy |
|:---|:---|:---|
| **Минимальный код для класса** | Всегда нужно объявлять конструктор, вызывающий `super()`. | Можно просто написать `class MyEx extends Exception {}` и пользоваться. |
| **Наследование конструкторов** | Требуется явно писать все конструкторы. | **`@InheritConstructors`** — автоматически наследует все конструкторы предка. |
| **Использование в `throw`** | Только оператор. | Выражение (можно использовать в `?:`, `?.` и т.д.). |
| **Типизация исключения в `catch`** | Только по имени класса. | Можно ловить по типу, а можно и по другим признакам (например, по сообщению в `case`). |

**5. Пример аннотации `@InheritConstructors` (Groovy-способ)**

Это самый идиоматичный способ создания исключений в Groovy.

```groovy
import groovy.transform.InheritConstructors

// Непроверяемое исключение
@InheritConstructors
class ValidationException extends RuntimeException {}

// Проверяемое исключение (формально, в Groovy разница не так критична)
@InheritConstructors
class BusinessException extends Exception {}

// Использование
throw new ValidationException("Имя пользователя не может быть пустым")
throw new BusinessException("Ошибка при обработке платежа")
```

**Резюме:** В Groovy создание исключений максимально упрощено благодаря аннотации `@InheritConstructors`, а оператор `throw` становится гибким выражением, что позволяет писать более выразительный и безопасный код.
